# AI Support For Student Projects And Capstones

Purpose: Add a project/capstone-focused section to the STEM teacher discussion.

## Core Point

AI can be especially valuable in student projects because projects require students to do many things they have not done before:

- pick a feasible idea
- understand unfamiliar tools
- install software
- configure hardware
- debug errors
- read documentation
- make tradeoffs
- test incomplete systems
- document progress
- recover from failure

This is where AI can help without simply doing the assignment. It can reduce friction and help students persist through the messy "how do I even start?" stage.

## Why This Matters

Capstone and project-based learning often fail for reasons that are not the core learning goal:

- students cannot get the environment set up
- they do not know which terms to search
- hardware or libraries are confusing
- tutorials are outdated
- installation errors stop progress
- students hit one error and stall
- teachers cannot individually debug every group's setup

AI can act as a first-line troubleshooting and planning assistant.

## Good AI Uses In Capstones

| Project Phase | AI Can Help With | Student Still Owns |
|---|---|---|
| Idea selection | Brainstorm feasible ideas and scope | Choosing a meaningful, realistic project |
| Requirements | Turn vague idea into goals, constraints, success criteria | Defining what success actually means |
| Planning | Break work into tasks, timeline, materials | Prioritizing and adapting the plan |
| Setup | Installation steps, configuration help, troubleshooting | Running commands, checking environment, documenting setup |
| Learning | Explain unfamiliar terms, libraries, circuits, APIs | Understanding enough to make decisions |
| Debugging | Interpret errors, suggest fixes, isolate causes | Testing fixes and explaining what worked |
| Design review | Identify risks, assumptions, missing parts | Deciding what to change |
| Documentation | Draft README, wiring notes, test plan, reflection prompts | Accuracy, clarity, final explanation |
| Presentation | Help organize demo narrative | Defending choices and evidence |

## Risks

AI can also weaken capstone learning if:

- it picks the project for the student
- it generates code the student cannot explain
- it creates a plan that is too ambitious
- it hides setup details that students need to understand
- it gives unsafe electronics or lab advice
- it invents library functions or outdated commands
- it becomes the only source of documentation

## Classroom Guardrails

For capstones, the rule should not be "no AI." A better rule:

"AI may be used as a project assistant, but the student/team must document what it helped with, verify technical claims, and explain the final system."

Required student evidence:

- project goal and constraints
- AI-use log
- setup notes
- error/debugging log
- test plan
- evidence of working prototype or partial prototype
- what AI got wrong or what the team rejected
- final explanation of how the system works

## Simple AI Use Log For Projects

Students maintain a short table:

| Date | What We Asked AI | What It Suggested | What We Tried | Result | What We Learned |
|---|---|---|---|---|---|

This turns AI use into visible learning instead of hidden assistance.

## Example: Raspberry Pi Project

Project: Build a Raspberry Pi sensor system that logs temperature and alerts when a threshold is crossed.

AI can help:

- list required parts
- explain GPIO pins
- suggest Python libraries
- help install packages
- interpret error messages
- draft a wiring checklist
- suggest test cases
- draft a README

Student must show:

- actual wiring diagram
- installation notes
- code they can explain
- test data
- threshold reasoning
- failure modes
- final demo

## Example: Robotics Project

Project: Improve autonomous robot behavior.

AI can help:

- brainstorm strategy
- compare sensors
- suggest pseudocode
- explain PID concepts
- review code structure
- suggest test cases
- help write a debugging checklist

Student must show:

- match/game constraints
- design tradeoffs
- sensor readings
- test results
- code walkthrough
- what changes improved performance

## Presentation Slide

Title: Capstones: Where AI Can Reduce Friction Without Replacing Learning

Bullets:

- Projects fail when students get stuck on setup, tools, errors, and unfamiliar documentation.
- AI can help students get unstuck and keep moving.
- The learning evidence must shift to logs, tests, decisions, and explanation.
- The student still owns the design, verification, and final defense.

## Discussion Prompt For Teachers

"In your project-based work, where do students get stuck: idea selection, setup, debugging, documentation, testing, or presentation?"

Follow-up:

"Where could AI help students persist, and what evidence would show they still understand the work?"

## Key Line

AI is useful in capstones because it can lower the barrier to starting. The teacher's job is to make sure it does not also lower the requirement to understand.
